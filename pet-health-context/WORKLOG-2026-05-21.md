## AI Unit Economics

Implemented a backend AI credit system so AI calls are no longer unmetered.

Key behavior:

- Users receive a small initial AI trial credit balance.
- AI features reserve credits before calling Gemini.
- Failed AI calls can refund reserved credits.
- Successful or failed attempts are logged for accounting.
- Credit ledger records credit movements.
- Rewarded ad credit claiming is available as an initial monetization hook.

11. Confirm AI credit balance updates from `2` to `3`.

Useful commands:

```bash
cd pet-health-frontend
yarn test:e2e --project=chromium
yarn playwright test --project=chromium --headed --workers=1
yarn playwright test --project=chromium --debug
PWDEBUG=1 yarn test:e2e --project=chromium
```

Note:

- The terminal used during verification was Git Bash (`MINGW64`), so `PWDEBUG=1 yarn ...` is the correct syntax there.
- PowerShell syntax is `$env:PWDEBUG="1"; yarn ...`.
- `PWDEBUG=1` pauses on each Playwright action and requires pressing Resume/Step in Playwright Inspector.

## Current Project Direction

The product strategy after today's work is:

- Free users should get meaningful value from Core Care without generating AI cost.
- AI features remain premium/scarce and are protected by credits, quality gates, and budget guards.
- Automation begins with web smoke E2E before expanding to more web scenarios or native app automation.

Likely next steps:

- Add more Web E2E coverage for health check and breed recognition with mocked AI responses.
- Add negative E2E tests for out-of-credit and invalid payload UX.
- Decide whether to create a PR from `feature/automation-tests`.
- Later, evaluate mobile native automation with WebdriverIO/Appium after web flows stabilize.
