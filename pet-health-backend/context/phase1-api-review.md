Below are several important technical notes to help you implement this file smoothly during Phase 1:1. IP Address Considerations (For Expo)
The documentation lists localhost:3000, but when you use a physical phone (like your iPhone 12 or 14) to run Expo Go, the phone will not recognize localhost as your computer.  Solution: You must use your computer's local IP address (e.g., [http://192.168.1.5:3000](http://192.168.1.5:3000)).  Tip: When running the Backend, tell Cursor: "Please log the local IP address of the machine when the server starts so I can easily copy it into the Expo URL structure."  2. Optimizing the AI Diagnosis API (Section 4)
To save costs and increase speed, you should perform Image Compression directly on the Mobile App before sending it to the Backend.  Use libraries like expo-image-picker or expo-camera combined with expo-image-manipulator to resize images to approximately 1000px - 1200px.  Reduce the quality to 0.7 - 0.8. This keeps the image file under 1MB while ensuring the AI can still read it clearly.  3. Synchronizing Data Structures between DB and AI
Since you are using Supabase, ensure that the column names in your Database (such as avatar_url, user_id) match the keys in the JSON returned by Gemini. This allows you to use the Supabase insert() command directly without having to map the data manually.  4. Handling Loading States
Calling Gemini 3 Flash typically takes 2–5 seconds, depending on the image size.  On the Mobile App, design a Skeleton effect or a Scanning Animation over the captured photo. This gives the user the feeling that the AI is actively "scanning" the wound or symptom.
